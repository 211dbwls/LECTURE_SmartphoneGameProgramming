XML, JSON, YAML은 컴퓨터가 서로 정보를 주고받을 때 사용하는 데이터를 쓰는 형식들을 말한다.

XML은 <tag/> 형식을 사용한다.
<shop/>과 같이 단일로 사용하기도 하고, <shop></shop>과 같이 시작과 종류 트리를 사용하기도 한다.
이때 shop은 태그명으로 각 데이터의 항목명이 된다.
<shop>과 </shop> 사이에는 순수 데이터 값 또는 다른 태그들이 중복으로 들어갈 수 있다. 
XML minifier을 사용하면 XML의  내용을 한 줄의 문자열로 바꿀 수 있고, XML formatter을 사용하면 minifier하기 전의 형태로  바꿀 수 있다.
XML의 장점은 주석 사용이 가능하다는 것과 요구된 구조화 형태를 제대로 갖췄는지 스키마를 통해 확인(xsd) 가능하다는 것이 있다. 

JSON은 자바 스크립트의 객체 표기법이다.
XML보다 좀 더 간결하며 {}으로 항목과 데이터를 구분한다.
객체 안에는 단일 데이터 또는 또 다른 데이터가 들어가고 []을 사용해 배열을 사용할 수도 있다.
JSON 역시 minifier을 사용해 한 불의 문자열로 바꿔줄 수 있다.
JSON은 문법 오류에 취약하다는 단점이 있다.

YAML은 데이터를 한 줄로 작성하지 않고 사람이 보기 편한 형식으로 작성하는 방식을 말한다.
줄바꿈과 태그가 필수로 사용된다.
장점으로는 주석과 상속이 가능하다는 것이 있다.
